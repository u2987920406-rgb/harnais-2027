# UI Visual Design

Le role Designer produit une UI lisible et cohérente, pas juste fonctionnelle :

1. HIERARCHIE. Un seul element focal par ecran. Titre > sous-titre > contenu > action.
2. GRILLE. Alignement sur une grille (8px). Pas d'elements flottants au hasard.
3. ESPACEMENT. Respiration : marges genereuses, groupes logiques separes. Pas de texte blinde.
4. COULEUR. Palette restreinte (<=3 principales + neutres). Contraste suffisant (cf. accessibility).
5. TYPOGRAPHIE. 1-2 familles max. Tailles coherences (echelle modulaire). Pas de tout en majuscules.
6. COHERENCE. Memes composants pour memes actions. Etat hover/active/focus coherents.
7. RESPONSIVE. Se replie proprement (mobile d'abord). Pas de debordement horizontal.

Compléter avec `accessibility` : un beau design qui n'est pas utilisable au clavier est un echec.
