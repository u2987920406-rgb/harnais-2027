# Internationalisation (i18n)

Les apps générées visent souvent FR et EN (Raf = français) :

1. SEPARER. Aucun texte d'UI en dur dans le JSX. Tout passe par une couche de traduction (clés).
2. CLES STABLES. clés descriptives (`home.title`), pas de chaines brutes comme clés.
3. FALLBACK. Si une traduction manque, fallback sur la langue par defaut (FR), jamais une clé crue.
4. FORMAT. Dates/nombres via Intl (locale-aware), pas de format maison.
5. CONTEXTE. Certains mots changent de genre/pluriel selon la langue : utiliser des fonctions de
   pluralisation, pas de concaténation naive.
6. BORNE. Ne pas sur-ingenierer : 2 langues (fr/en) d'abord. Ajouter une locale = ajouter un fichier.

Le Verifier peut refuser une UI avec du texte en dur dans le composant si le brief demande i18n.
